# Content requirements

## GitHub README

Include project overview, use cases, features, actual tech stack, Mermaid architecture when useful, folder tree, installation, usage, safety model, results, lessons learned, and future improvements.

## Medium article

Write 1,500–3,000 Chinese words as a chronological technical story. Include at least five title options, opening problem, background, debugging sequence, final solution and why it works, architecture/tool/workflow/AI lessons, pitfalls, conclusion, SEO title, meta description, URL slug, and relevant tags. Do not copy README prose.

## Portfolio collateral

Create:

- three to five STAR-style resume bullets;
- an approximately 300-Chinese-character LinkedIn project introduction;
- a Conventional Commit message;
- a PR description with Summary, Changes, Testing, Screenshots, and Future Work;
- concrete follow-on projects aligned with AI Software Engineer, AI Solution Architect, and AI Agent Consultant growth.

Keep reusable publishing copy in `docs/portfolio.md` and the complete Medium draft in `docs/medium-article.md`.
